Q: magellan-client.jar? Shouldn't the file better be called magellan.exe?

A: No. Basically, a jar-file is a renamed zip file with some additional information needed by the java virtual machine to start the jar-file. You should not decompress the file. After installing the JRE, a doubleclick on magellan-client.jar should start Magellan.

 

Q: Why does another program or tool (winzip e.g.) start when I double click on Magellan?

A: The file-extension *.jar is not associated with Java, but with another program (like Winzip). In the tools Powerarchiver and WinZip you can undo this setting, if not, you may follow these steps:

  1. Select 'RUN' in the Start menu.
  2. Enter 'regedit' and press 'Enter'.
  3. Open the entry 'HKEY_CLASSES_ROOT' in the tree at the left side and select '.jar'.
  4. Doubleclick on '(Default)' in the right side.
  5. Enter 'jarfile' (without the single quotes) and select 'OK'.
  6. Close regedit - ready.

 

Q: Why doesn't Magellan start when I click on the magellan-client.jar?

A: One possible reason might be that the directory path to magellan-client.jar contains wide spaces. This is not an error in Magellan but an unlucky connection between .jar-files and Java.

The simple solution is to move Magellan into another -better- directory. The other way is this:

  1. Select 'RUN' in the Start menu.
  2. Enter 'regedit' and press 'Enter'.
  3. Open the entry 'HKEY_CLASSES_ROOT' in the tree at the left side and select 'jarfile'.
  4. Open the subtrees for 'shell', 'open' and finally 'command'.
  5. Doubleclick right on '(Default)', something like (Path to Java)\javaw.exe -jar %1 should be visible.
  6. Replace %1 with "%1"
  7. Close regedit - ready.

Background: if the location of Magellan is the directory C:\My Documents\magellan-client.jar, then the entry in regedit results in a call to java like (Path to Java)\javaw.exe -jar C:\My Documents\magellan-client.jar, that means, java tries to start the file C:\My with parameter Documents\magellan-client.jar - not good. After our little change it will look like this: (Path to Java)\javaw.exe -jar "C:\My Documents\magellan-client.jar", so file magellan-client.jar is started for sure.

 

Q: How do I set up a link between CR-Files and Magellan in Windows 2K, XP?

A: For the file type "CR" a new setting is needed. You can achive this in the file explorer: Extra->Folder Options->Filetypes:

  1. Choose the file type "CR".
  2. Select the button "Advanced".
  3. Choose the button "New" in the frame "edit file type".
  4. Enter the value "Magellan" (without quotes) into the field "process ".
  5. Enter the value "java.exe" -Xmx256M -jar "magellan-client.jar" "%1" (including all quotes) into the field "program for this process ". Java.exe and magellan-client.jar need to have a complete path (e.g. "c:\games\eressea\magellan-client.jar"). If this path contains wide spaces, the double quotes are badly needed. For the meaning of the "-Xmx 256M" see below.
  6. Choose the button "OK"
  7. In window "edit file type ", choose the entry "Magellan" from the list "processes".
  8. Choose the option "as standard". The entry Magellan should now be formatted as bold.
  9. Ready :-)

 

Q: How to start Magellan via the command line?

A: Magellan (and Java) often show their error messages in the command window only. To check these messages it is useful to start Magellan from the command line.

Open a command window (menu start->run and enter command and click OK). Now enter the command javaw -Xmx256M -jar "(Path to Magellan)\magellan-client.jar" , e.g. javaw -Xmx256M -jar "C:\My Documents\magellan-client.jar".

 

Q: How to run Magellan on a Mac?

A: Unfortunately, Mac OS X was the first Mac-OS including a java version under which Magellan is running. To load Magellan follow the command line instructions: command line.

Additionally it is useful to work with a rules file specially converted for Mac. (Source unknown, sry). If you got the file, save it to a folder named "rules" and point a ressource-path to the parent directory. After restarting Magellan, the new files should be in use and the possible income in a region should be calculated correctly now.

 

Q: I have enough RAM, why does Magellan report memory errors?

A: Usually, Java uses only 64MB of RAM, regardless of how much you really have. With some parameters it is possible to increase the amount of memory which is available for Magellan and is managed by the JVM:

Windows:
Create a connection to Magellan, open properties (with right-click). Edit the "Target" line as follows (use your directory paths where needed):
javaw.exe -Xms128M -Xmx512M -jar "c:\Program Files\Eressea\Magellan\magellan-client.jar" or
"C:\Program Files\Java\jre6\bin\javaw.exe" -Xms128M -Xmx512M -jar "c:\Program Files\Eressea\Magellan\magellan-client.jar"

Linux (shell, in the magellan directory):
java -Xms128M -Xmx512M -jar magellan-client.jar

The Java-VM is told now to acquire a minimum of 128MB and a maximum of 512MB RAM. Depending on your machine you can play around with the values. It does not usually hurt to tell java to use all your memory as a maximum. There will still be enough memory for your other programs.

 

Q: Why does Magellan stop using Base36 numbers for units?

A:In the CR the tag 36;Basis is probably missing. It defines the base-number for Magellan. Eressea-CRs usually use 36, other PBeMs or CR-Versions (e.g. Verdanon) work with decimal numbers (Base 10) and do not know this tag. Magellan assumes Base 10 if no other information is present in the CR.
