Q: magellan.jar? Should the file better named magellan.exe?

A:No. Basicly a jar-File is a renamed zip File with some additional information needed by the Java Virtual Machine to start the jar-File. You should not decompress the file. After installing the JRE a doubleclick on magellan.jar should start Magellan.

 

Q: Why another program or tool (winzip e.g.) is starting, when double clicking on Magellan?

A: The File-ending *.jar is not connected with Java, but with another program (like Winzip). In the tools Powerarchiver and WinZip you can Undo this setting, if not, you may follow these steps:

  1. Select in menu Start 'RUN'
  2. Enter 'regedit' and press 'Enter'
  3. Open in the three at the left side the entry 'HKEY_CLASSES_ROOT' and select '.jar'
  4. Doubleclick in the right side '(Default)'
  5. Enter 'jarfile' (without the ' signs) and select OK
  6. Close regedit - ready

 

Q: Why Magellan isn´t starting, when I click on the magellan.jar?

A: One possible reason might be, that the directory path to magellan.jar contains wide spaces. This is not an error in Magellan but an unlucky connection betwenn .jar-files and Java.

The simple solution is to move Magellan into another -better- directory. The other way is this:

  1. Select in menu Start 'RUN'
  2. Enter 'regedit' and press 'Enter'
  3. Open in the three at the left side the entry 'HKEY_CLASSES_ROOT' and select 'jarfile'
  4. Open the subtrees for 'shell', 'open' and finally 'command'
  5. Doubleclick right on '(Default)', something like (Path to Java)\javaw.exe -jar %1 should be visible.
  6. Replace %1 with "%1"
  7. Close regedit - ready

Background: Location of Magellan is directory C:\My Documents\magellan.jar , than the entry in regedit results a call to java like (Path to Java)\javaw.exe -jar C:\My Documents\magellan.jar, that means, java tries to start the file C:\My with parameter Documents\magellan.jar - not good. After our little change it will look another way: (Path to Java)\javaw.exe -jar "C:\My Documents\magellan.jar", so file magellan.jar is started for sure.

 

Q: How do I built a Link between CR-Files and Magellan in Windows 2K, XP?

A: For the file type "CR" a new setting is needed. You can achive this in the file explorer: Extra->Folder Options->Filetypes:

  1. Choose file type "CR"
  2. Select Button "Advanced"
  3. Choose in the Frame "edit file type" the button "New"
  4. Enter in field "process " value ' Magellan 'ein
  5. Enter in field "program for this process " the value ' "java.exe" -jar "magellan.jar" "%1" ' Java.exe and magellan.jar need to have a complete path(e.g. "c:\games\eressea\magellan.jar"). Contains this path wide spaces, the " are badly needed.
  6. Choose the button "OK"
  7. Choose in window "edit file type " the entry "Magellan" from the list "processes" .
  8. Choose the Button "as standard". The entry Magellan should now be formated as bold.
  9. Ready :-)

 

Q: How to start Magellan via the command line?

A: Magellan (and Java) often showing error messages in the command window only. To check these messages it is useful to start Magellan from the command line.

Open a command window (menu start->run and enter command and click OK). Now enter the command javaw -jar "(Path to Magellan)\magellan.jar" , e.g. javaw -jar "C:\My Documents\magellan.jar".

Q: I have enough RAM, why Magellan reports memory errors?

A: With some parameters it is possible, to increase the amount of memory which is available for Magellan and is managed by the JVM:

Windows:
Create a Connection to Magellan, open Properties (with right-click). Edit the "Target" line as follows (use your directory paths were needed):
C:\WINDOWS\javaw.exe -Xms128MB -Xmx256MB -jar c:\programs\Eressea\Magellan\magellan.jar

Linux (shell):
java -Xms128MB -Xmx256MB -jar magellan.jar

The Java-VM is told to acquire now min. 128MB and max. 256MB RAM. Depending on your machine you can play around with the values.

 

Q: Why Magellan stops using Base36 numbers for units?

A:In the CR the tag 36;Basis is probably missing. It defines the base-number for Magellan. Eressea-CRs usually use 36, other PBeMs usng oder (or other) CR-Versions (e.g. Verdanon) working with decimal numbers(Base 10) and do not know this tag. Magellan assumes Base 10, if no other information in the CR is present
